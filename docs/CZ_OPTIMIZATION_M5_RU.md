# Оптимизация зон консолидации (M5) по парам

Этот документ дает готовый рабочий план подбора параметров:
- `CZ_LookbackN`
- `CZ_ATR_K`

Цель: получить стабильное формирование зон консолидации без переизбытка ложных входов.

---

## 1) Базовая логика (коротко)

В EA зона формируется, когда:

`range(high-low за CZ_LookbackN баров) <= CZ_ATR_K * ATR`

Практический смысл:
- больше `CZ_LookbackN` -> зона реже;
- меньше `CZ_ATR_K` -> зона реже;
- слишком мягко (`Lookback` низкий + `ATR_K` высокий) -> много "шумных" зон и ложных сигналов;
- слишком жестко (`Lookback` высокий + `ATR_K` низкий) -> "засуха" по зонам.

---

## 2) Стартовые якорные значения (уже близки к рабочим)

- `EURUSD`: `Lookback=14`, `ATR_K=1.35`
- `GBPUSD`: `Lookback=14`, `ATR_K=1.30`
- `USDJPY`: `Lookback=18`, `ATR_K=1.15` (`EnableSignalB=false`)
- `USDCHF`: `Lookback=17`, `ATR_K=1.10` (`EnableSignalB=false`)

---

## 3) Shortlist для прогона в Strategy Tester

Для каждой пары прогонять сначала **только приоритетные 5 точек**, затем расширять при необходимости.

### EURUSD (M5)

Приоритет:
1. `14 / 1.35` (база)
2. `14 / 1.25` (чуть жестче)
3. `12 / 1.35` (чуть чаще зоны)
4. `16 / 1.30` (стабилизация шума)
5. `12 / 1.25` (компромисс)

Что сразу не брать:
- `Lookback <= 10` с `ATR_K >= 1.6` (шумно),
- `Lookback >= 20` с `ATR_K <= 1.0` (слишком редко).

### GBPUSD (M5)

Приоритет:
1. `14 / 1.30` (база)
2. `14 / 1.20`
3. `12 / 1.30`
4. `16 / 1.20`
5. `12 / 1.20`

Что сразу не брать:
- `Lookback <= 10` + `ATR_K >= 1.5`,
- `Lookback >= 18` + `ATR_K <= 1.0`.

### USDJPY (M5, без сигнала B)

Приоритет:
1. `18 / 1.15` (база)
2. `18 / 1.05` (жестче качество)
3. `16 / 1.15` (чуть больше частота)
4. `20 / 1.10` (антишум)
5. `16 / 1.05` (компромисс)

Что сразу не брать:
- `ATR_K >= 1.4` (часто дает избыток слабых зон),
- `Lookback >= 22` + `ATR_K <= 1.0` (дефицит сигналов).

### USDCHF (M5, без сигнала B)

Приоритет:
1. `17 / 1.10` (база)
2. `17 / 1.00`
3. `15 / 1.10`
4. `19 / 1.05`
5. `15 / 1.00`

Что сразу не брать:
- `Lookback <= 12` + `ATR_K >= 1.4`,
- `Lookback >= 21` + `ATR_K <= 0.95`.

---

## 4) Как сравнивать результаты (единые правила)

Сначала отсев:
- `trades_per_month < 8` -> недостаточная частота;
- `profit_factor < 1.10` -> слабое качество;
- `expectancy <= 0` -> отбраковка.

Далее выбор финала:
1. Максимальный `PF` **не приоритет**, если низкая частота.
2. Лучше сет с чуть меньшим PF, но стабильным числом сделок.
3. Проверять равномерность по месяцам: без явных "пустых" провалов.
4. Для JPY/CHF не включать `Signal B`, пока A+C не дадут устойчивость.

---

## 5) Пошаговый порядок прогона (чтобы не тратить лишнее время)

1. По каждой паре прогнать 5 приоритетных комбинаций.
2. Оставить топ-2 по сочетанию `PF + expectancy + частота`.
3. Сделать форвард на свежем отрезке (не из оптимизации).
4. Принять итоговый сет только если форвард не ломает статистику.

---

## 6) Практический вывод

Если задача — "зоны формируются регулярно, но без лишнего шума", то оптимальный старт:
- EURUSD/GBPUSD: держаться в диапазоне `Lookback 12..16`, `ATR_K 1.2..1.35`;
- USDJPY/USDCHF: держаться в диапазоне `Lookback 15..20`, `ATR_K 1.0..1.15`, с выключенным `Signal B`.

Это дает баланс между частотой и качеством и соответствует текущей архитектуре EA.
