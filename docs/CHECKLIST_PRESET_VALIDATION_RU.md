# Чек-лист проверки новых пресетов (MT5 Strategy Tester)

Документ нужен, чтобы принимать решение по новым `.set` одинаково и без "на глаз".

Использовать для сравнения:
- **базовый (старый) пресет**: `MQL5/Presets/Acteck_v1.09_<PAIR>.set`
- **новый пресет**: `tools/run_01/generated_v2/Acteck_v1.09_<PAIR>.set`

Проверка выполняется **отдельно по каждой паре**: `EURUSD`, `GBPUSD`, `USDJPY`, `USDCHF`.

---

## 1) Подготовка теста (обязательно одинаковая для старого и нового)

- [ ] Один и тот же символ и таймфрейм.
- [ ] Один и тот же период теста (даты начала/конца).
- [ ] Одинаковые настройки спреда/модели/депозита/комиссий.
- [ ] Одинаковые торговые ограничения (если задаются в тестере).
- [ ] Сначала прогон старого пресета, потом нового.
- [ ] Результаты обоих прогонов сохранены (отчёт/скрин/экспорт).

Если условия не совпадают, сравнение считается недействительным.

---

## 2) Метрики, которые нужно выписать после каждого прогона

Для старого и нового пресета заполнить:

- [ ] Net Profit
- [ ] Profit Factor
- [ ] Expected Payoff / Expectancy
- [ ] Recovery Factor (если есть)
- [ ] Max Drawdown (absolute/relative)
- [ ] Total Trades
- [ ] Win Rate (%)
- [ ] Average win / Average loss (если есть)
- [ ] Серия убыточных сделок (max consecutive losses)

---

## 3) Правила решения “ПРИНЯТЬ / ОТКЛОНИТЬ”

Новый пресет можно считать рабочим, если выполнены все условия:

- [ ] `Profit Factor` не ниже старого и желательно > 1.10.
- [ ] `Expectancy` не ниже старого и > 0.
- [ ] `Max Drawdown` не вырос существенно (ориентир: не более +10–15% относительно старого).
- [ ] Количество сделок не стало слишком маленьким (ориентир: не менее 70% от старого).
- [ ] Нет явного ухудшения по серии убыточных сделок.

Если хотя бы 1–2 ключевые метрики заметно ухудшились (PF/Expectancy/DD), пресет отклоняется.

---

## 4) Быстрый фильтр перед детальным анализом

Если видите одно из условий ниже, пресет сразу в “кандидаты на отклонение”:

- [ ] `Profit Factor < 1.0`
- [ ] `Expectancy <= 0`
- [ ] Резкий рост просадки
- [ ] Сделок слишком мало (стратегия “замолчала”)

---

## 5) Порядок внедрения (важно)

- [ ] Не обновлять сразу все пары в реале.
- [ ] Сначала выбрать 1 пару с лучшими метриками.
- [ ] Запустить на демо/микро-период.
- [ ] Только после подтверждения — переносить остальные пары.

---

## 6) Шаблон итогового решения по паре

Скопируйте блок для каждой пары:

```text
PAIR: EURUSD
Старый пресет: MQL5/Presets/Acteck_v1.09_EURUSD.set
Новый пресет:  tools/run_01/generated_v2/Acteck_v1.09_EURUSD.set

PF: old=__  new=__
Expectancy: old=__  new=__
Drawdown: old=__  new=__
Trades: old=__  new=__
WinRate: old=__  new=__

Решение: ПРИНЯТЬ / ОТКЛОНИТЬ
Причина: __
```

---

## 7) Минимально разумные ориентиры

Это не жёсткие правила, а практические ориентиры:

- PF: целиться в `> 1.10`, хорошо `> 1.20`.
- Expectancy: обязательно `> 0`.
- Drawdown: лучше снижать, а не увеличивать.
- Сделок должно хватать для статистики; очень маленькая выборка = высокий риск ошибки.

---

## 8) Важное ограничение

Полностью убрать убыточные сделки невозможно.  
Цель — не “0 losses”, а **устойчиво положительное матожидание + контролируемая просадка**.
