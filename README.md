# Acteck v 1.09 (MT5)

Практический проект полуавтоматического/автоматического советника для MetaTrader 5 с портфельными пресетами и контролируемым циклом адаптации параметров по истории сделок.

Репозиторий включает:
- советник `Acteck v 1.09.mq5` для компиляции в `.ex5`;
- готовые пресеты для `EURUSD / GBPUSD / USDJPY / USDCHF`;
- инструменты анализа и "самообучения" (offline recalibration) через Python;
- one-click `.command`-запуск конвейера `HTML -> CSV -> пересчёт пресетов -> копирование в MT5`.

---

## 1) Что это за алгоритм

Стратегия построена вокруг **зон консолидации** и трёх сигналов:
- `A` — пробой зоны;
- `B` — ложный пробой (возврат в диапазон по тени/телу);
- `C` — ретест после пробоя.

Сигналы дополнительно фильтруются EMA/ATR/спредом/ограничениями риска и правилом сопровождения позиции (BE/partial close/ограничения времени удержания и пр.).

### Ключевые цели
- убрать заведомо слабые входы;
- адаптировать пороги под конкретную валютную пару;
- поддерживать стабильный процесс регулярной переоценки пресетов без "чёрного ящика" внутри EA.

## 3) Как работает анализ и "самообучение"

Важно: это **не online-ML внутри терминала**. 

Подход осознанно сделан как контролируемый offline-процесс:
1. Вы выгружаете историю сделок из MT5 (HTML/CSV).
2. Скрипт переводит историю в чистый CSV-формат.
3. Скрипт recalibration оценивает статистику по парам и корректирует диапазон параметров.
4. Генерируются новые `.set` по символам.
5. Новые пресеты копируются в `MQL5/Profiles/Presets` и проходят тест/форвард.

Преимущество: прозрачность и воспроизводимость. Можно всегда откатиться к базовому пресету.

## 4) Параметры, которые адаптируются

В версии v2 фокус на параметрах входной логики и фильтрах, например:
- `CZ_LookbackN`
- `CZ_ATR_K`
- `BreakCloseOffset`
- `RetestDepth`
- `WickRatio`
- `ATR_Min`

Плюс формируются дополнительные артефакты:
- `recalibration_report_v2.json`
- `adaptive_profile_v2.json`

Они нужны для аудита: почему скрипт предложил именно такие значения.

## 5) One-click workflow (рекомендованный)

### Шаг A. Экспорт из MT5
`Toolbox -> History -> Save as Report (HTML)`

### Шаг B. Запуск `.command`
`tools/run_mt5_pipeline_v2.command`

Скрипт делает:
1. `mt5_html_to_csv.py`
2. `recalibrate_presets_v2.py`
3. копирование `Acteck_v1.09_*.set` в папку пресетов MT5.

### Шаг C. Применение в терминале
На каждом символе загрузить соответствующий `.set` через `Load`.

---

## 6) Рекомендуемая частота переоценки

- 1 раз в 1–2 недели;
- окно истории 3–6 месяцев;
- новый пресет брать в работу только после проверки:
  - достаточно сделок (`min-trades`),
  - profit factor не хуже базы,
  - нет резкого падения частоты торговых дней.

---

## 7) Компиляция `.ex5`

`.ex5` создаётся только в MetaEditor:
1. скопировать `.mq5` в `MQL5/Experts/`;
2. открыть в MetaEditor;
3. `Compile (F7)`;
4. получить `Acteck v 1.09.ex5`.

Название советника в навигаторе терминала соответствует имени файла: **Acteck v 1.09**.

---

## 8) Практические замечания по риску

- На разных парах одна и та же логика даёт разную устойчивость; поэтому pair-specific пресеты обязательны.
- Изменения всегда сначала гонять на демо/в тестере.
- Скрипты адаптации не должны использоваться как замена риск-менеджмента (лот, лимит одновременных позиций, контроль новостей и т.д.).
