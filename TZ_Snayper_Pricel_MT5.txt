Техническое задание
Проект: Советник MT5 «Снайпер — Прицел (аналог)»
Платформа: MetaTrader 5, тип счёта — hedging (поддержка частичного закрытия)
Инструменты/ТФ: любые, по умолчанию EURUSD M5 (смена через параметры)
Цель: реализовать в советнике логику ТС по мотивам «Снайпер — Прицел» с базовым набором модулей. Визуализация на графике обязательна.

1) Объём работ (базовая версия)
1.1 Реакционные зоны + визуализация
  • Зона консолидации детектируется, если за N баров диапазон (High–Low) ≤ K * ATR(P). Параметры N, K, P — настраиваемые.
  • Ширина зоны = макс/мин за Lookback; зона рисуется прямоугольником до инвалидации (пробой по закрытию > BreakCloseOffset пунктов).
  • Подсветка актуальной «реакционной» зоны (граница предполагаемой реакции) — цвет/стиль настраиваются.

1.2 Ключевые уровни дня (автопостроение, визуализация)
  • HOD/LOD (High/Low of Day) — по смене суток по времени брокера (параметр DayStartHour).
  • Баланс-день — середина между HOD и LOD (опция: вместо середины использовать цену открытия дня).
  • Параметры: показывать уровни текущего/предыдущего дня; длина отображения; подписи.

1.3 Сигналы входа (по структуре свечей + положению относительно зон/уровней)
  Реализуются три сценария (каждый можно включать/отключать):
  A) Пробой зоны: бар закрывается за пределами зоны на BreakCloseOffset пунктов; вход на следующем баре в сторону пробоя.
  B) Возврат (ложный пробой): тень за уровнем > WickRatio * Body, а закрытие — внутри зоны; вход в сторону возврата.
  C) Retest после пробоя: после A цена возвращается в зону на глубину RetestDepth и формирует подтверждение «закрытие в направлении пробоя»; вход по рынку.
  Доп. подтверждение «структурой свечей» (опц.): engulfing/внешний бар (выбор в параметрах).

1.4 Фильтры направления
  • EMA-trend: вход только если цена/Close сигнального бара в нужной стороне от EMA_Period (режим в параметрах).
  • ATR-волатильность: запрет входа при ATR ниже ATR_Min или при диапазоне бара ниже RangeMin = m * ATR.
  • Все параметры — настраиваемые.

1.5 Торговый блок
  • Вход — рыночный; режим «только сигналы без входа» (алерты/стрелки) — опция.
  • SL/TP: фикс в пунктах или от границы зоны (+смещение) — выбор источника в параметрах.
  • Breakeven, TrailingStop, PartialClose (по объёму/ступеням) — включаемые опции.
  • Защита: MaxSpread, Slippage, «одна сделка на бар», ReEntries (повторные входы), MaxPositions — параметры.
  • Комментарии/Code: MagicNumber, префикс имён объектов на графике — параметры.

1.6 Индикатор «проторговки»
  • Линия, соединяющая последовательные свинг-экстремумы текущего движения.
  • Свинги: ZigZag‑подобная логика по ZZ_Depth/Deviation/Backstep или фракталы (режим выбирается параметром).
  • Цвет/толщина/показ — настраиваются.

1.7 Подсветка точки входа
  • Прямоугольная зона/маркер на свече входа с подписью Buy/Sell и отображением цены входа/SL/TP; стрелка — опционально.
  • История сигналов (не только активные сделки) — опция.

1.8 Совместимость MT5 (hedging)
  • Корректная работа с ордерами, частичным закрытием и учётом нескольких позиций по символу.

1.9 Режим перерасчёта (детерминированность результатов)
  • По умолчанию — «только на закрытии бара» (детект и вход происходят после закрытия).
  • Опция «внутри бара»: пересчёт условий раз в N секунд; по умолчанию выключено (во избежание шума).

Важное замечание: реализация «по мотивам» стратегии и по формализованным правилам ТЗ; привязки к закрытым индикаторам/коду третьих лиц нет. Допустима небольшая визуальная разница, не влияющая на логику.

2) Визуализация/интерфейс
  • Объекты: уровни/зоны/подписи/стрелки/прямоугольники; пользовательская панель не обязательна.
  • Настройка цветовой схемы; порядок отрисовки (чтобы зоны не закрывали свечи).
  • Уведомления: журнал, Alert, Push — переключатели.

3) Входные параметры (минимум)
  Общие: Timeframe, DayStartHour, UsePrevDayLevels, ShowLevelsLenBars.
  Зоны: CZ_LookbackN, CZ_ATR_Period, CZ_ATR_K, BreakCloseOffset, RetestDepth, WickRatio, ConfirmPattern(Off/Engulfing/OutsideBar).
  Фильтры: EMA_Period, EMA_FilterMode(Price/Close/BarClose), ATR_Period, ATR_Min, RangeMinATR_Mult.
  Торговля/риск: LotMode(Fixed/PercentRisk), Lot/Percent, SL_Mode(Fixed/FromZone), SL_Points/SL_Offset, TP_Mode(Fixed/FromZone), TP_Points/TP_Offset,
                 UseBE, BE_Trigger, BE_Offset, UseTS, TS_Start, TS_Step, PartialClose_On, PC_Step1/Vol1, PC_Step2/Vol2, OnlyOneTradePerBar,
                 ReEntries, MaxPositions, MaxSpread, Slippage, MagicNumber, CommentPrefix.
  Проторговка/свинг: SwingMode(ZigZag/Fractals), ZZ_Depth, ZZ_Deviation, ZZ_Backstep, ShowSwingLine.
  Графика/уведомления: ShowZones, ShowEntryMarker, Цвета…, Alerts(OnScreen/Push/Off).

4) Критерии приёмки
  4.1 Уровни и баланс‑день — корректно обновляются при смене суток; опция предыдущего дня работает.
  4.2 Зоны — формируются при соблюдении условий п.1.1; инвалидация по закрытию с учётом BreakCloseOffset.
  4.3 Сигналы A/B/C — появляются строго по правилам п.1.3; при режиме «только закрытие» — после закрытия; внутри‑бар — только при включении.
  4.4 Фильтры EMA/ATR — при включении блокируют входы, нарушающие тренд/минимальную волатильность.
  4.5 Торговый блок — SL/TP/BE/TS/PartialClose работают по параметрам; на hedging‑счёте частичное закрытие выполняется корректно.
  4.6 Проторговка — линия строится по выбранному режиму свингов, без «прыжков» при пролистывании истории.
  4.7 Подсветка входа — прямоугольник/стрелка, подписи цены/SL/TP.
  4.8 Тестер — визуальный режим без ошибок/варнингов; результаты повторяемы при одинаковых входных.
  4.9 Логи — информативные сообщения о входах/выходах/отклонениях (напр., спред > MaxSpread).

5) Не входит в базовую версию (можно отдельным этапом)
  • Автоуровни по М1–H1 и сессионные уровни (Asia/London/New York) с объединением микроуровней.
  • Расширенные фильтры (угол EMA, мульти‑EMA, набор свечных комбинаций, фильтр времени/дней недели, новостной фильтр).
  • Мультисимвольная торговля из одного графика; сетка/усреднение/мартингейл.
  • Полная графическая панель управления.

6) Передаваемые материалы
  • EA.ex5 (для проверки), EA.mq5 (исходник после приёмки), пример .set.
  • Краткая инструкция (txt/pdf) с описанием параметров.

7) Сроки и стоимость (по сервису MQL5)
  • Стоимость: 300 USD.
  • Срок: до 10 рабочих дней на первую версию после старта сделки на сервисе.
  • Доработки в рамках ТЗ — по необходимости, сроки фиксируются в переписке сервиса.

8) Прочие условия
  • Различия в котировках брокеров/тайминге тиков могут приводить к отличию отдельных сигналов — это не считается дефектом.
  • По умолчанию сигналы и входы формируются на закрытии бара (детерминированность). Внутри‑бар — опционально.
